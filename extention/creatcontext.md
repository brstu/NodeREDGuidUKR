# Створення власних вузлів 

## [Node context](https://nodered.org/docs/creating-nodes/context)

Вузол може зберігати дані у своєму контекстному об'єкті.

Для отримання додаткової інформації про контекст читайте [Робота з контекстом](../base/1_7.md).

У вузлі доступні три області контексту:

- Node - видимий лише той вузол, який встановив значення
- Flow - видимий для всіх вузлів одного потоку (або вкладки в редакторі)
- Global - видимий для всіх вузлів

На відміну від вузла функції, який забезпечує заздалегідь задані змінні для доступу до кожного з цих контекстів, користувацький вузол повинен отримати доступ до цих контекстів для себе наступним чином:

```javascript
// Доступ до об'єкту контекст з вузла
var nodeContext = this.context();

var flowContext = this.context().flow;

var globalContext = this.context().global;
```

Кожен з цих контекстних об'єктів має однакові функції `get` /` set`, описані в [Робота з вузлом Function](../base/1_5.md).

Примітка. Конфігураційні вузли, які використовуються іншими спільними вузлами та які вони спільно використовують, за замовчуванням є глобальними, якщо інше не вказано користувачем вузла. Тому не можна вважати, що вони мають доступ до контексту потоку.
